## 7.1 标准训练流水线（PyTorch模板）

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader

# ===== 1. 数据准备 =====
train_dataset = MyDataset(train=True, transform=train_transform)
train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True)

# ===== 2. 模型 =====
model = MyModel(num_classes=10).to(device)
criterion = nn.CrossEntropyLoss()                   # 多分类
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=epochs)

# ===== 3. 训练循环 =====
for epoch in range(num_epochs):
    model.train()          # 训练模式→Dropout生效, BN用batch统计
    train_loss = 0
    for x, y in train_loader:
        x, y = x.to(device), y.to(device)
        optimizer.zero_grad()          # ① 清空梯度
        output = model(x)              # ② 前向传播
        loss = criterion(output, y)    # ③ 计算损失
        loss.backward()                # ④ 反向传播（计算梯度）
        optimizer.step()               # ⑤ 更新参数
        train_loss += loss.item()
    scheduler.step()       # ⑥ 更新学习率

    # ===== 4. 验证 =====
    model.eval()           # 评估模式→Dropout关闭, BN用全局统计
    correct = 0
    with torch.no_grad():  # 不计算梯度→省显存
        for x, y in val_loader:
            x, y = x.to(device), y.to(device)
            output = model(x)
            pred = output.argmax(dim=1)
            correct += (pred == y).sum().item()
    acc = correct / len(val_dataset)
    print(f'Epoch {epoch}: loss={train_loss:.4f}, acc={acc:.4f}')
```

### 每一步的解释

| 步骤 | 做什么 | 常见错误 |
|------|--------|----------|
| `zero_grad()` | 清空上一轮的梯度 | 忘记写→梯度累加（或有意累积） |
| `model(x)` | 前向传播，算出预测 | — |
| `criterion(output,y)` | 算损失（预测和真实的差距） | 输出已做Softmax再传CrossEntropy |
| `loss.backward()` | 反向传播，算出每个参数的梯度 | 忘了先算loss |
| `optimizer.step()` | 用梯度更新参数 | 梯度累积时忘了等accumulation步 |

### ◆ 自测 ◆

**Q1.** 以下代码有什么问题？

```python
for x, y in loader:
    output = model(x)
    loss = criterion(output, y)
    loss.backward()
    optimizer.step()
```

**Q2.** `model.train()` 和 `model.eval()` 影响哪些层？具体怎么影响？

**Q3.** `with torch.no_grad()` 是必须的吗？不写会怎样？

---
<details>
<summary>答案</summary>
<p><strong>A1.</strong> 缺少 <code>optimizer.zero_grad()</code>。梯度会不断累加，每个batch的梯度都是之前所有batch梯度之和→相当于batch_size越来越大→可能导致爆炸或不稳定。</p>
<p><strong>A2.</strong> Dropout：train时随机失活，eval时关闭。BatchNorm：train时用当前mini-batch的均值和方差（同时更新全局running mean/var），eval时用训练期间累积的全局统计量。</p>
<p><strong>A3.</strong> 不是必须的（代码不会报错），但不写会浪费显存。<code>no_grad</code>下PyTorch不构建计算图→不需要存储中间变量用于反向传播→省显存。验证时不需要梯度，建议始终加上。</p>
</details>

## 7.3 常见代码错误排查

### 错误1：双重Softmax

```python
# ❌
output = F.softmax(model(x), dim=1)  # 模型输出已经过了Softmax
loss = F.cross_entropy(output, y)     # CrossEntropy内部又做了一次Softmax!

# ✅
output = model(x)                      # 原始logits
loss = F.cross_entropy(output, y)
```

**症状**：loss不下降，始终在-log(1/C)附近（如10分类≈2.3）

### 错误2：view不连续

```python
# ❌
x = x.transpose(1, 2).view(B, -1)     # transpose改变stride→不连续→view报错

# ✅
x = x.transpose(1, 2).contiguous().view(B, -1)
# 或直接用reshape
x = x.transpose(1, 2).reshape(B, -1)
```

### 错误3：忘记清空梯度

### 错误4：验证时不设eval()

### 错误5：Sigmoid隐藏层导致梯度消失

```python
# ❌ 深网络
x = torch.sigmoid(self.fc1(x))   # σ'(x)≤0.25→深网络梯度消失
x = torch.sigmoid(self.fc2(x))

# ✅
x = F.relu(self.fc1(x))
x = F.relu(self.fc2(x))
```

---
<details>
<summary>答案</summary>
<p><strong>A1.</strong> ①双重Softmax（模型输出了Softmax又传入CrossEntropyLoss→概率趋于均匀→loss≈-log(1/10)=2.3026）；②学习率太大导致模型发散，输出始终混乱。</p>
<p><strong>A2.</strong> transpose改变了tensor的内存布局（stride不一致），view需要连续内存。当transpose破坏了连续性时就会报错。解决：加 <code>.contiguous()</code> 或用 <code>.reshape()</code>。</p>
</details>

## 7.4 参数量与计算量速算

### 参数量

| 层 | 公式 |
|----|------|
| Linear(in, out) | in×out + out(bias) |
| Conv2D(in, out, K) | in×out×K² + out(bias) |
| LSTM(in, hidden) 单层 | 4×(in×h + h×h + h) |
| Embedding(V, D) | V×D |
| BatchNorm(C) | 2C |
| LayerNorm(D) | 2D |

### 特征图尺寸

$$ H_{out} = \lfloor (H_{in} - K + 2P) / S \rfloor + 1 $$

## 7.5 完整代码分析例题

### 例题

```python
class PoetryModel(nn.Module):
    def __init__(self, vocab_size=5000, embed_dim=100, hidden_dim=128):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, embed_dim)
        self.lstm = nn.LSTM(embed_dim, hidden_dim, num_layers=2,
                            batch_first=True, dropout=0.3)
        self.mlp = nn.Sequential(
            nn.Dropout(0.3),
            nn.Linear(hidden_dim, 512),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(512, vocab_size)
        )
        self.log_softmax = nn.LogSoftmax(dim=-1)

    def forward(self, x, is_test=False):
        batch_size = x.size(0)
        embed = self.embed(x)
        h0 = torch.zeros(2, batch_size, 128, device=x.device)
        c0 = torch.zeros(2, batch_size, 128, device=x.device)
        out, _ = self.lstm(embed, (h0, c0))
        if is_test:
            out = out[:, -1, :]
        out = self.mlp(out)
        return self.log_softmax(out)
```

**Q1.** 这是什么网络？用于什么任务？

**Q2.** 模型用了哪些正则化技术？

**Q3.** h0的第一个维度为什么是2？

**Q4.** is_test=True 时 `out[:, -1, :]` 的作用？

**Q5.** 为什么输出用 LogSoftmax 而不是 Softmax？配合什么损失函数？

**Q6.** 计算参数量：Embedding层 + LSTM层(无bias)

---
<details>
<summary>答案</summary>
<p><strong>A1.</strong> LSTM语言模型（唐诗生成）。输入字符序列→预测下一个字符。</p>
<p><strong>A2.</strong> LSTM层间Dropout(0.3)、MLP中两层Dropout(0.3)。共3处正则化。</p>
<p><strong>A3.</strong> num_layers=2，每层LSTM都需要最初的h0和c0。形状(2, B, 128) = (层数, batch, hidden_dim)。</p>
<p><strong>A4.</strong> 测试/生成时只取最后一个时间步的输出来预测下一个字符（前面的输出不需要了）。</p>
<p><strong>A5.</strong> LogSoftmax + NLLLoss = CrossEntropyLoss。数值更稳定（在log空间计算）。配合NLLLoss使用。</p>
<p><strong>A6.</strong> Embedding: 5000×100=500K。LSTM(无bias): 4×2×(100×128+128×128) = 8×(12800+16384) = 8×29184 = 233,472。</p>
</details>

## 本章要点速记

| 项目 | 要点 |
|------|------|
| 训练循环 | zero_grad→forward→loss→backward→step |
| model.eval() | Dropout关 + BN用全局统计 |
| torch.no_grad() | 推理时不构建计算图 |
| view vs reshape | view需要连续内存，reshape不需要 |
| 双重Softmax | 最常见bug→loss≈-log(1/C)不下降 |
| 参数量 | Conv: in×out×K²+bias; Linear: in×out+bias; LSTM: 4×(ih+hh+bias) |
