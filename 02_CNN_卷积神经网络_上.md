## 2.1 为什么需要CNN？

DNN处理图像的问题：
1. Flatten后参数量爆炸（第一层可达数亿）
2. 丢失空间结构（相邻像素关系被破坏）
3. 无平移不变性（物体移动后要重新学）

CNN通过**局部连接+权重共享**解决这三个问题。

## 2.2 卷积运算

### 本质

一个 $K \times K$ 的卷积核在图像上滑动，每一步与覆盖区域做**逐元素乘法再求和**，得到特征图上一个值。

### 输出尺寸公式（必考）

$$ H_{out} = \left\lfloor \frac{H_{in} - K + 2P}{S} \right\rfloor + 1 $$

| 符号 | 含义 | 常用值 |
|------|------|--------|
| $H_{in}$ | 输入尺寸 | — |
| $K$ | 卷积核大小 | 3（最常用） |
| $P$ | Padding圈数 | 1（same padding: P=(K-1)/2） |
| $S$ | Stride步长 | 1（保持尺寸）或 2（下采样） |

### 参数量

$$ \text{Params}_{conv} = C_{in} \times C_{out} \times K^2 + C_{out} \ (\text{bias}) $$

### 卷积核在干什么？

不同卷积核学习不同的**特征检测器**：
- 某些核检测水平边缘
- 某些核检测垂直边缘
- 深层核检测纹理、形状、物体部件
- 更深层的核检测完整物体

### ◆ 自测 ◆

**Q1.** 输入32×32×3, Conv2D(3, 64, 3, padding=1, stride=1) → 输出尺寸？参数量？

**Q2.** 输入224×224, Conv2D(3, 64, 7, stride=2, padding=3) → 输出尺寸？

**Q3.** 为什么常用 3×3 卷积而很少用 7×7 或 11×11？

**Q4.** "same padding" 是什么意思？padding 应该设为多少？

---
<details><summary>答案</summary>

**A1.** 输出: (64, 32, 32)（padding=1保持尺寸）。参数量: 3×64×9+64=1728+64=1792。

**A2.** (224-7+6)/2+1 = 111.5→111? 不对，(224-7+2×3)/2+1 = 223/2+1=111+1=112。所以是 112×112。

**A3.** 2层3×3卷积的感受野=5×5，参数量=2×3×3×C²；1层5×5卷积的感受野=5×5，参数量=5×5×C²。3×3用更少的参数实现相同感受野，且中间多了一个ReLU增加非线性。3层3×3感受野=7×7，参数远少于1层7×7。

**A4.** "same padding"指输出尺寸=输入尺寸。由公式 H = (H-K+2P)/S+1 = H → P = (K-1)/2（当S=1时）。K=3→P=1, K=5→P=2, K=7→P=3。
</details>

## 2.3 池化（Pooling）

### 作用

下采样——减小特征图尺寸，降低计算量，同时保留重要信息。

### 类型

| 池化 | 操作 | 特点 |
|------|------|------|
| **MaxPool2D(2,2)** | 2×2窗口取最大值 | 保留最显著特征，常用 |
| **AvgPool2D(2,2)** | 2×2窗口取平均值 | 保留整体信息 |
| **AdaptiveAvgPool2D(H,W)** | 强制输出为H×W | 接受任意输入尺寸 |
| **GlobalAvgPool** | 整张特征图→1个值 | 替代Flatten，无参数！ |

### 池化的关键性质

- **无学习参数**（纯粹是数学运算）
- **提供平移不变性**（最大池化对微小位移不敏感）
- MaxPool2D(2,2)：尺寸减半

### ◆ 自测 ◆

**Q1.** MaxPool2D 和 AvgPool2D 各有什么优势？什么时候用哪个？

**Q2.** Global Average Pooling 相比 Flatten + Linear 有什么优势？

**Q3.** 为什么池化层没有参数也能工作？

---
<details><summary>答案</summary>

**A1.** MaxPool：保留最显著特征（如最亮的边缘），适合分类任务。AvgPool：保留整体信息，更平滑，适合一些分割任务。CNN分类中MaxPool是默认选择。

**A2.** ①零参数→不会过拟合；②输出维度=通道数（固定），不依赖输入尺寸→可以处理任意大小的输入图像。Flatten+Linear会随输入尺寸变化。

**A3.** 因为它是固定的数学操作（取最大/取平均），不需要学习。就像 ReLU 也没有参数一样。
</details>

## 2.4 CNN典型架构模式

### 标准CNN块

```
Conv2D → BatchNorm → ReLU → MaxPool
```

这个模式在大多数CNN中反复出现。各组件的作用：

| 组件 | 作用 |
|------|------|
| Conv2D | 提取特征 |
| BatchNorm | 归一化→稳定训练→允许更大学习率 |
| ReLU | 引入非线性 |
| MaxPool | 下采样→减小计算量→扩大感受野 |

### 通道数变化的规律

通常通道数逐层翻倍，空间尺寸逐层减半：

```
输入:     (3,   224, 224)
Conv1:    (64,  112, 112)    ← C翻倍，H/W减半
Conv2:    (128,  56,  56)
Conv3:    (256,  28,  28)
Conv4:    (512,  14,  14)
Conv5:    (512,   7,   7)
```

通道数增加 → 提取更多种特征；空间尺寸减小 → 越到深层越关注全局语义。

### ◆ 自测 ◆

**Q1.** 经过3次 MaxPool2D(kernel=2, stride=2)：224→?→?→?

**Q2.** BatchNorm 应该放在 Conv 和 ReLU 的什么位置？

**Q3.** 为什么靠后的卷积层通道数通常比前面的多？

---
<details><summary>答案</summary>

**A1.** 224→112→56→28。每次减半。

**A2.** Conv → BN → ReLU。先卷积得出特征→BN归一化稳定分布→ReLU激活。如果顺序是ReLU→BN或Conv→ReLU→BN效果通常会变差。

**A3.** 浅层通道少（检测基础特征：边缘→3或64通道足够）；深层通道多（检测复杂语义：物体部件/纹理→需要256/512通道表达丰富的语义信息）。这是经验规律，也受计算约束影响。
</details>

## 2.5 ResNet 残差网络

### 核心创新：跳跃连接（Skip Connection）

$$ H(x) = F(x) + x $$

- $F(x)$ 是卷积块学到的"残差"（输入与目标的差值）
- $+x$ 是跳跃连接（把输入直接加到输出）

### 数学解释（为什么有效）

反向传播时：
$$ \frac{\partial H}{\partial x} = \frac{\partial F}{\partial x} + \mathbf{1} $$

这个 **+1** 确保了：即使 ∂F/∂x 非常小（趋近于0），梯度也至少为1，不会消失。梯度可以沿残差连接**无障碍**传播到浅层。

### Bottleneck 结构

```
1×1 Conv (降维: 256→64) → 3×3 Conv (特征提取) → 1×1 Conv (升维: 64→256)
```

**为什么这样设计？** 如果直接做 256→256 的3×3卷积，参数量 = 256×256×9≈590K。Bottleneck的参数量 = 256×64 + 64×64×9 + 64×256 ≈ 68K。参数量减少到约1/8！

### ResNet18 vs ResNet50

| | ResNet18 | ResNet50 |
|---|----------|----------|
| 块类型 | BasicBlock(2个3×3Conv) | Bottleneck(1×1→3×3→1×1) |
| 层数 | 18 | 50 |
| 参数量 | ~11M | ~23M |

### ◆ 自测 ◆

**Q1.** ResNet 的"残差"（Residual）指的是什么？

**Q2.** 如果去掉跳跃连接，ResNet 会退化成什么？

**Q3.** 为什么 ResNet 可以堆叠到 152 层，而 VGG 最深只有 19 层？

---
<details><summary>答案</summary>

**A1.** 指 F(x) = H(x) - x，即网络学习的是"输出减去输入"的残差部分，而非直接学习 H(x)。跳跃连接把简单的恒等映射 x 直接传过去，网络只需学增量部分。

**A2.** 退化成普通CNN（如VGG风格的堆叠卷积）。去掉跳跃连接也失去梯度高速通道，深层时梯度消失严重。

**A3.** ResNet的跳跃连接让梯度可以直通浅层（∂H/∂x = ∂F/∂x + 1），即使深层梯度极小也不会消失到0。VGG中梯度必须经过每一层的卷积+BatchNorm+ReLU逐层回传，连乘导致更深的层无法训练。
</details>

## 2.6 VGGNet

### 核心哲学

**全部使用 3×3 卷积核**（相对当时流行的11×11等大核）。

- 2层 3×3 → 感受野 = 5×5（替代1层5×5）
- 3层 3×3 → 感受野 = 7×7（替代1层7×7）

用小核堆叠代替大核：参数量更少、非线性更多（每层都有ReLU）。

### 结构模式

```
[2×3×3Conv]→Pool  →  [2×3×3Conv]→Pool  →  [3×3×3Conv]→Pool
   64→64      ↓        128→128      ↓       256→256→256   ↓
             56                  28                      14

[3×3×3Conv]→Pool  →  [3×3×3Conv]→Pool  →  FC(4096)→FC(4096)→FC(1000)
512→512→512   ↓     512→512→512   ↓
              7                     (Flatten=512×7×7)
```

5个卷积块 + 3个全连接层 = **VGG16**（13层Conv + 3层FC）。

## 2.7 迁移学习

### 核心思想

在大数据集（如ImageNet-1000类、1400万张）上预训练的模型，其浅中层已学到通用的视觉特征（边缘、纹理、形状、物体部件），可用到其他图像任务上。

### 两种策略

| 策略 | 做法 | 适用 |
|------|------|------|
| **Feature Extractor** | 冻结预训练权重，只训练最后的分类头 | 小数据（<1000张） |
| **Fine-tuning** | 预训练权重初始化，全模型用更小lr微调 | 中等数据+任务相关 |

### 代码

```python
# 加载预训练ResNet18并修改输出层
model = models.resnet18(pretrained=True)
model.fc = nn.Linear(model.fc.in_features, num_classes)  # 换分类头

# Fine-tuning：小学习率训练全模型
optimizer = Adam(model.parameters(), lr=1e-4)  # 比从头训练小10-100倍
```

### ◆ 自测 ◆

**Q1.** 预训练模型输入是224×224，但我的数据是32×32。怎么办？

**Q2.** 什么情况下迁移学习效果反而不好？

**Q3.** `pretrained=True` 时 PyTorch 从哪里下载权重？

---
<details><summary>答案</summary>

**A1.** 三种方案：(i)把输入Resize到224×224再训练（最简单，但增加了计算量）；(ii)修改模型第一层stride让它适配32×32（需要调整后续层尺寸）；(iii)保留预训练权重但调整第一层卷积→其余层用作初始化。

**A2.** 预训练数据（自然图像）和目标数据差异极大时（如医学影像、卫星图、红外图、深度图等），预训练权重学到的纹理和颜色特征可能不适用。此时从头训练或领域内预训练更好。

**A3.** 从PyTorch官方服务器自动下载到 `~/.cache/torch/hub/checkpoints/`。首次需要网络连接。
</details>
