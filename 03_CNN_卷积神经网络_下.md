## 3.1 1×1 卷积

### 不只是"点卷积"

1×1 卷积在空间维度上只覆盖1个像素（无空间信息提取），但它在**通道维度**上做全连接：

$$ y_{h,w,c_{out}} = \sum_{c_{in}=1}^{C_{in}} w_{c_{in}, c_{out}} \cdot x_{h,w,c_{in}} $$

### 两大用途

1. **改变通道数**（升维/降维）：64通道→128通道，或256通道→64通道
2. **跨通道信息融合**：整合不同通道的特征（类似通道间的全连接）

> ResNet Bottleneck 用 1×1 降维是深度学习中最经典的工程优化之一。

### ◆ 自测 ◆

**Q1.** 1×1卷积和Linear层有什么本质区别？

**Q2.** Conv2D(256, 64, 1) 的参数量是多少？

---
<details><summary>答案</summary>

**A1.** 1×1卷积作用在(H,W)每个空间位置独立执行相同的通道变换（共享权重），输出仍是(H,W,C_out)。Linear将整个张量展平后全连接，输出是1维向量。1×1保持空间结构，Linear破坏它。

**A2.** 256×64×1×1 + 64 = 16,384 + 64 = 16,448。
</details>

## 3.2 空洞卷积（Dilated / Atrous Convolution）

### 概念

在卷积核的像素之间插入"空洞"（间距），不增加参数的情况下增大感受野。

```
普通卷积 3×3 (dilation=1):   空洞卷积 3×3 (dilation=2):
  ■ ■ ■                        ■ _ ■ _ ■
  ■ ■ ■    感受野=3×3          _ _ _ _ _    感受野=5×5
  ■ ■ ■                        ■ _ ■ _ ■
                               _ _ _ _ _
                               ■ _ ■ _ ■
```

### 等效卷积核大小

$$ K_{eff} = K + (K-1)(d-1) $$

dilation=2, K=3 → K_eff = 3+2×1 = 5

### 为什么需要？

在不做池化（不缩小特征图，保留空间精度）的情况下增大感受野。语义分割任务中非常重要（需要精细的空间定位 + 大范围的上下文信息）。

### ◆ 自测 ◆

**Q1.** dilation=3, kernel=3 时，等效感受野是多少？

**Q2.** 空洞卷积和普通卷积+MaxPool哪种更适合语义分割？为什么？

---
<details><summary>答案</summary>

**A1.** K_eff = 3 + (3-1)×(3-1) = 3 + 2×2 = 7×7。

**A2.** 空洞卷积更适合。语义分割需要像素级输出，MaxPool会永久丢失空间精度；空洞卷积在不缩小特征图的前提下获得大感受野，保留空间分辨率。
</details>

## 3.3 深度可分离卷积

### 两步

```
普通Conv2D(3,128,3):
  每个输出通道 = 所有输入通道的3×3卷积之和  → 3×3×3×128=3456次乘法

Depthwise Separable Conv:
  Step1(Depthwise): 每个输入通道独立做3×3卷积     → 3×3×3=27次乘法
  Step2(Pointwise): 1×1卷积融合通道(3→128)      → 3×128=384次乘法
  总计: 27+384=411次乘法 (约为普通卷积的1/8!)
```

### 代表模型：MobileNet

完全基于深度可分离卷积，适合移动端和嵌入式设备。

## 3.4 Dropout 正则化

### 原理

训练时每个神经元以概率 $p$ 被随机"丢弃"（输出置0）。测试时全部神经元参与。

```python
self.dropout = nn.Dropout(p=0.5)  # 训练时50%的神经元被丢弃
```

**解释**：每次训练看到的是不同的"子网络"→相当于训练了指数级数量的网络→测试时集成它们。

### 重要细节

- 训练时（model.train()）：**生效**，随机丢弃
- 测试时（model.eval()）：**关闭**，全部参与，PyTorch自动处理缩放
- 通常 $p$：CNN 用 0.3~0.5，Transformer 用 0.1

### 放在哪里？

```
Conv → BN → ReLU → Dropout → ... → Linear → ReLU → Dropout → Linear
```

通常放在激活函数之后、下一层之前。RNN中可以在层间加Dropout（`nn.LSTM(..., dropout=0.3)`）。

### ◆ 自测 ◆

**Q1.** Dropout=0.5时，训练和测试的输出期望为什么是一致的？

**Q2.** Dropout 和 BatchNorm 能同时使用吗？有冲突吗？

**Q3.** 为什么CNN中 Dropout 常用在 FC 层而不是 Conv 层？

---
<details><summary>答案</summary>

**A1.** 训练时保留的神经元输出会乘以 1/(1-p)=2 进行放大，使得期望不变。测试时不做Dropout也不缩放，因为期望和训练时一致。PyTorch内部自动完成这个缩放。

**A2.** 可以同时使用，但存在"方差偏移"（Variance Shift）问题。BN在训练和测试时统计量不同，Dropout又改变了神经元分布，可能导致训练和测试不匹配。实践中很多模型仍一起用。建议顺序：Conv→BN→ReLU→Dropout。

**A3.** Conv层的参数量少（共享权重），天然有一定的正则化效果，过拟合风险低。FC层参数量极大（占网络总参数的80-90%），过拟合风险高，所以Dropout主要用在FC层。
</details>

## 3.5 数据增强

### 为什么需要？

过拟合的本质：模型记住训练样本的"无关细节"（背景色、物体位置等）。数据增强通过随机变换迫使模型学到**不变性**。

### 图像增强方法

| 方法 | 代码 | 作用 |
|------|------|------|
| 随机水平翻转 | `RandomHorizontalFlip(0.5)` | 左右翻转不变性 |
| 随机旋转 | `RandomRotation(10)` | 旋转不变性 |
| 颜色抖动 | `ColorJitter(brightness=0.2)` | 光照不变性 |
| 随机裁剪 | `RandomResizedCrop(224)` | 缩放/位置不变性 |
| 随机擦除 | `RandomErasing()` | 遮挡鲁棒性 |

### Mixup / CutMix（高级增强）

```python
# Mixup：两张图像按比例混合（输入+标签都混合）
x_mix = λ * x_a + (1-λ) * x_b
y_mix = λ * y_a + (1-λ) * y_b

# CutMix：将一张图的一块区域粘贴到另一张图上
```

这些方法能大幅提升小数据集的泛化能力。

### ◆ 自测 ◆

**Q1.** 数据增强在训练和测试时都需要吗？

**Q2.** 竖直翻转（VerticalFlip）适合什么任务？不适合什么任务？

---
<details><summary>答案</summary>

**A1.** 只有训练时需要。测试时用原始图像，不做增强。数据增强的目的是增加训练数据多样性，测试时不需要。

**A2.** 适合：卫星图、显微镜图（无"上下"概念）。不适合：自然场景分类（天空通常在上方，上下翻转会破坏语义）。
</details>

## 3.6 CNN 设计原则总结

### 选多大卷积核？

| 核大小 | 适合 | 例子 |
|--------|------|------|
| 3×3 | **首选**，堆叠获得大感受野 | VGG, ResNet |
| 1×1 | 通道变换、Bottleneck降维 | ResNet Bottleneck |
| 5×5, 7×7 | 第一层快速降低空间分辨率 | ResNet首层(7×7, stride2) |
| 11×11 | 已过时 | AlexNet |

### 通道数如何设计？

- 逐层翻倍：64→128→256→512
- 或用 1×1 Bottleneck 内部降维

### 什么时候加 Pool / stride>1？

- 每 2-3 个 Conv 后做一次下采样
- 空间尺寸逐步减小，通道数逐步增加
- 最终到 7×7 或 4×4 时接 FC 或 GAP

### ◆ 自测 ◆

**Q1.** 以下哪种做法是错的？
A. Conv→BN→ReLU
B. Conv→ReLU→BN
C. BN→Conv→ReLU
D. Conv→Pool→ReLU

**Q2.** 为什么现在第一层Conv通常用 stride=2 替代单独的 Pool 层来下采样？

---
<details><summary>答案</summary>

**A1.** B和C都违背常规。B把BN放ReLU后（BN应该归一化Conv的输出，不是ReLU后的）。C把BN放Conv前（BN应该放在Conv之后）。D中Pool一般不放在ReLU前面。正确答案是A：Conv→BN→ReLU。

**A2.** 带stride的卷积是可学习的下采样（有参数），MaxPool是固定的无参数操作。stride-conv能同时完成特征提取和下采样，更灵活。ResNet首层就用的 stride=2 的7×7卷积，不用单独MaxPool。
</details>
