    深度学习高频考点 通用问答¶

    全网高频考题合集 — 概念解释 + 公式 + 代码 + 对比例表
    每个知识点先讲后练，答案在每节末尾，学习时遮挡答案即可自测

    一、基础概念¶

    1.1 深度学习 vs 传统机器学习¶

    维度 传统ML (SVM/决策树等) 深度学习
    特征提取 手工设计（需领域专家） 自动学习（端到端）
    数据需求 中小规模 大规模
    计算资源 CPU 即可 GPU/TPU
    可解释性 较好 较差（黑盒）
    典型场景 表格数据、小样本 图像、语音、NLP

    1.2 神经网络的基本组成¶

    神经元: z = Σ(w_i·x_i) + b  →  a = f(z)
网络 = 多层神经元堆叠
训练 = 前向传播(算输出) + 反向传播(算梯度) + 梯度下降(更新参数)

    1.3 权重和偏置¶

    $$ z = Wx + b $$

    • 权重(W)：控制每个输入的重要程度，学习的是特征之间的关联模式
    • 偏置(b)：平移激活函数的阈值，让模型更灵活

    1.4 Epoch / Batch / Iteration¶

    概念 定义 示例(10000样本, batch=100)
    Batch 一次前向/反向传播用的样本 100个样本
    Iteration 完成一个 batch 的训练=一次参数更新 1 iteration = 100样本
    Epoch 全部训练数据被用一遍 1 epoch = 100 iterations

    ◆ 自测题 ◆¶

    Q1. 为什么深度学习需要大量数据？

    Q2. 一个 epoch 包含多少次 iteration？公式是什么？

    Q3. "batch_size 越大训练越快"这个说法对吗？为什么？

    答案

    A1. 深度学习模型参数量巨大（百万~千亿级），需要大量数据才能学到泛化能力强的特征，否则容易过拟合（记住训练集而非学习规律）。

    A2. iterations_per_epoch = ceil(训练样本总数 / batch_size)。如10000样本，batch=128，则约为79次 iteration。

    A3. 不完全对。大 batch 每次更新时梯度估计更准确（并行效率高），但可能泛化性能下降（容易陷入尖锐的局部最优）。小 batch 噪声大但有正则化效果，泛化可能更好。

    二、激活函数¶

    2.1 为什么需要激活函数？¶

    没有激活函数，多层网络等价于单层线性变换： $$ W_2(W_1 x) = (W_2 W_1) x = W' x $$ 激活函数引入非线性，使网络能拟合任意复杂函数。

    2.2 常用激活函数对比¶

    函数 公式 输出范围 优点 缺点
    Sigmoid σ(x)=1/(1+e⁻ˣ) (0,1) 平滑，输出可解释为概率 梯度消失(σ'≤0.25)，非零均值
    Tanh (eˣ-e⁻ˣ)/(eˣ+e⁻ˣ) (-1,1) 零均值比Sigmoid好 仍有梯度消失
    ReLU max(0,x) [0,+∞) 计算简单，缓解梯度消失 Dead ReLU(x<0→梯度永远0)
    LeakyReLU max(0.01x,x) (-∞,+∞) 解决Dead ReLU 负区间斜率需人工设定
    GELU x·Φ(x) (-∞,+∞) Transformer首选 计算稍复杂
    Softmax eˣⁱ/Σeˣʲ (0,1),和为1 输出概率分布 仅用于输出层

    2.3 梯度消失的数学解释¶

    Sigmoid 导数：σ'(x) = σ(x)(1-σ(x))，最大值 = 0.25

    反向传播链式求导：∂L/∂W₁ = ∂L/∂aₙ · σ'(zₙ) · Wₙ · σ'(z_{n-1}) · W_{n-1} · ... · σ'(z₁)

    每层乘一次 ≤0.25 的因子，N 层后梯度 ≈ 0.25^N → 指数衰减 → 前面几层几乎不学习。

    ◆ 自测题 ◆¶

    Q1. 为什么现在几乎不在隐藏层使用 Sigmoid/Tanh？

    Q2. 什么是"Dead ReLU"？如何解决？

    Q3. 输出层做二分类用激活，多分类用激活，回归用___激活。

    Q4. 如果网络很深，以下哪项组合最容易导致梯度消失？ A. ReLU + BatchNorm B. Sigmoid + 无正则化 C. LeakyReLU + 残差连接 D. GELU + LayerNorm

    答案

    A1. Sigmoid/Tanh 在饱和区导数接近0，深层网络中梯度逐层衰减为0（梯度消失），导致训练缓慢甚至停滞。ReLU 正区间导数为1，梯度可以无损传播。

    A2. Dead ReLU：某些神经元在训练中总是输出0（输入始终为负），梯度为0，权重永不更新。解决：使用 LeakyReLU/PReLU；使用 Kaiming 初始化；降低学习率。

    A3. Sigmoid(二分类)，Softmax(多分类)，无/Linear(回归)。

    A4. B。Sigmoid 导数最大0.25，深网络连乘后梯度几乎消失；无BatchNorm/残差连接更恶化此问题。

    三、损失函数¶

    3.1 损失函数速查¶

    损失函数 公式 适用任务
    MSE (1/n)Σ(y-ŷ)² 回归
    MAE (1/n)Σ y-ŷ
    BCE -[y·log(ŷ)+(1-y)·log(1-ŷ)] 二分类
    CrossEntropy -Σ y_c·log(ŷ_c) 多分类
    NLLLoss -log(ŷ_class) 配合LogSoftmax

    3.2 交叉熵详解¶

    # 手动实现交叉熵（帮助你理解考试代码）
def cross_entropy(logits, labels):
    """
    logits: (N, C) 模型原始输出
    labels: (N,)   int型标签 [0, C-1]
    """
    # Step 1: Softmax 将 logits 转为概率
    exp_logits = np.exp(logits - np.max(logits, axis=1, keepdims=True))
    probs = exp_logits / np.sum(exp_logits, axis=1, keepdims=True)
    # Step 2: 取正确类别的负对数似然
    N = labels.shape[0]
    correct_probs = probs[np.arange(N), labels]
    loss = -np.mean(np.log(correct_probs + 1e-8))
    return loss

    3.3 关键易错点¶

    Q: PyTorch 的 CrossEntropyLoss 内部包含 Softmax 吗？

    答：包含！ nn.CrossEntropyLoss() = LogSoftmax + NLLLoss

    ❌ 错误做法（双重Softmax，输出接近均匀分布，模型学不会）： 

    output = F.softmax(model(x), dim=1)  # 多此一举！
loss = F.cross_entropy(output, y)

    ✅ 正确做法： 

    output = model(x)   # 模型输出原始logits
loss = F.cross_entropy(output, y)

    ◆ 自测题 ◆¶

    Q1. 为什么回归不能用交叉熵？分类不能用MSE？

    Q2. PyTorch中以下两段代码的loss是否相等？ 

    # (A)
loss = F.cross_entropy(logits, labels)
# (B)
loss = F.nll_loss(F.log_softmax(logits, dim=1), labels)

    Q3. Softmax 函数中减去 np.max(x) 的目的是什么？

    Q4. 对于二分类问题，使用 BCEWithLogitsLoss 和 CrossEntropyLoss(num_classes=2) 有区别吗？

    答案

    A1. 回归输出连续值，交叉熵要求输入是概率分布(0~1, 和为1)；分类的标签是离散的，MSE 对概率差异不敏感且梯度小(配合Sigmoid时容易梯度消失)。

    A2. 相等（浮点误差范围内）。CrossEntropyLoss 内部就是 LogSoftmax + NLLLoss。

    A3. 防止指数溢出：当 x 很大时 e^x 会溢出为 inf，减去max后最大值变为0，e^0=1，不会溢出。这是数值稳定性技巧，不影响 Softmax 结果（因为分子分母同时缩小）。

    A4. 数学上等价，但实现不同。BCEWithLogitsLoss 使用单个 Sigmoid 输出一个概率值。CrossEntropyLoss(num_classes=2) 使用两个 logits + Softmax。二分类通常用前者（更高效）。

    四、优化器¶

    4.1 梯度下降变体¶

    优化器 核心思想 公式（简化）
    SGD 沿负梯度方向更新 θ = θ - η·∇L
    SGD+Momentum 加入动量（惯性），加速收敛 v = βv + ∇L; θ = θ - η·v
    Adam 动量 + 自适应学习率 m=β₁·m+(1-β₁)·∇L; v=β₂·v+(1-β₂)·(∇L)²; θ=θ-η·m/(√v+ε)
    AdamW Adam + 解耦权重衰减 与Adam相同但 L2 正则化直接作用于θ

    4.2 各优化器适用场景¶

    场景 推荐优化器
    图像分类（CNN） Adam / SGD+Momentum
    Transformer/NLP AdamW
    小数据集 Adam（自适应，无需精细调参）
    追求SOTA SGD+Momentum（精细调参后泛化最好）

    4.3 学习率调度¶

    # StepLR: 每step_size个epoch将lr乘以gamma
scheduler = StepLR(optimizer, step_size=30, gamma=0.1)

# Cosine: 余弦曲线平滑衰减（常用）
scheduler = CosineAnnealingLR(optimizer, T_max=epochs)

# ReduceLROnPlateau: 指标停滞时降低lr
scheduler = ReduceLROnPlateau(optimizer, mode='min', patience=10)

# Warmup: 训练初期逐步增大lr到目标值（Transformer必备）

    4.4 学习率的影响¶

    • 太大：震荡、发散、loss 变成 NaN
    • 太小：收敛太慢、陷入局部最优
    • Warmup 必要性：训练初期模型参数随机，梯度大且不稳定，先从小 lr 开始逐步增大可避免模型跑偏

    ◆ 自测题 ◆¶

    Q1. Adam 相比 SGD 有什么优势？为什么很多时候还是用 SGD？

    Q2. 训练到后期发现 loss 震荡不下降，可能是什么原因？如何解决？

    Q3. Transformer 训练为什么通常需要 Warmup？

    Q4. 学习率从 0.01 改变为 0.001，以下哪个说法正确？ A. 训练一定变慢 B. 训练一定更稳定 C. 可能导致收敛更慢但更稳定 D. 没有影响

    答案

    A1. Adam 自适应调节每个参数的学习率，对超参数不敏感，收敛快。但 SGD 在精心调参后泛化性能往往更好（平坦的最优解）。

    A2. 可能原因：①学习率太大→震荡；②batch_size太小→梯度噪声大。解决：降低学习率或使用学习率衰减、增大batch_size、使用动量或Adam。

    A3. Transformer 训练初期梯度方差大，不加Warmup直接使用大学习率会导致模型跑偏（梯度信号不稳定）。Warmup让模型先"找到方向"，再加速收敛。

    A4. C。

    五、正则化与防过拟合¶

    5.1 Dropout¶

    • 训练时：每个神经元以概率 p 被"丢弃"（输出置0），其余神经元输出乘以 1/(1-p) 或推理时不做缩放
    • 测试时：全部神经元参与，通常不需要缩放（PyTorch自动处理）
    • 原理：每次训练相当于训练一个"子网络"，测试时相当于多个子网络的集成（Bagging 效果）
    # PyTorch示例
self.dropout = nn.Dropout(p=0.5)  # 50%概率丢弃
# model.train()时生效，model.eval()时自动关闭

    5.2 Batch Normalization¶

    $$ \hat{x} = \frac{x - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} $$ $$ y = \gamma \hat{x} + \beta $$ (γ和β是可学习参数，允许恢复原始分布)

    作用： 1. 加速训练（可以用更大的学习率） 2. 缓解梯度消失/爆炸（输入分布稳定） 3. 轻微正则化效果（mini-batch的噪声）

    5.3 BatchNorm vs LayerNorm¶

    BatchNorm LayerNorm
    归一化维度 每个特征在 batch 维度 每个样本在 feature 维度
    batch_size 依赖 强依赖（小batch效果差） 不依赖
    适用的模型 CNN Transformer, RNN
    序列长度可变 不友好 友好

    5.4 过拟合 vs 欠拟合¶

    过拟合 欠拟合
    训练集表现 很好（loss极低） 差（loss高）
    验证集表现 差（比训练集差很多） 差（和训练集差不多）
    原因 模型太复杂/数据太少 模型太简单/训练不够
    解决 加数据/正则化/Dropout/早停 加模型容量/减正则/多训练

    5.5 其他正则化技术¶

    技术 原理
    L1正则 损失加 λ
    L2正则(Weight Decay) 损失加 λW²，防止权重过大
    早停(Early Stopping) 验证loss不再下降时停止训练
    数据增强 对训练数据做随机变换增加多样性
    Label Smoothing 标签从[0,0,1,0]变[0.025,0.025,0.925,0.025]，防过度自信

    ◆ 自测题 ◆¶

    Q1. Dropout 在训练时乘以 1/(1-p) 的目的是什么？

    Q2. 为什么 CNN 用 BatchNorm，而 Transformer 用 LayerNorm？

    Q3. train_loss=0.05, val_loss=0.8，这属于什么情况？如何处理？

    Q4. 以下哪些是正则化技术？（多选） A. Dropout B. BatchNorm C. 数据增强 D. 增大batch_size E. Early Stopping F. 梯度裁剪

    答案

    A1. 因为训练时丢弃了比例为p的神经元，剩余神经元的输出期望只占原来的(1-p)。乘以1/(1-p)保证训练和测试时输出的期望一致（或等价地在测试时乘以(1-p)）。PyTorch 默认在 eval 模式不做缩放，而是在训练时做缩放。

    A2. CNN 处理图像(B,C,H,W)——batch维度大，channel维度固定，BN 在 batch 维度归一化有效。Transformer 处理序列(B,L,D)——序列长度可变，BN 在序列维度统计不稳定；LN 对每个样本独立归一化，不受batch和序列长度影响。

    A3. 严重过拟合。解决方法：①加 Dropout/L2 正则；②数据增强；③减小模型（层数/神经元数）；④早停；⑤获取更多数据；⑥增大batch_size（引入更多噪声）。

    A4. A, B, C, E 都是。D 增大 batch 可能减少噪声反而降低正则化效果。F 梯度裁剪是防梯度爆炸的训练技巧，不直接防过拟合。

    六、梯度消失与爆炸¶

    6.1 问题本质¶

    BPTT(通过时间反向传播) 中，梯度跨时间步连乘： $$ \frac{\partial L}{\partial h_1} \approx \frac{\partial L}{\partial h_T} \prod_{t=2}^T \frac{\partial h_t}{\partial h_{t-1}} $$

    若 |∂h_t/∂h_{t-1}| < 1 → 连乘 → 0（梯度消失） 若 |∂h_t/∂h_{t-1}| > 1 → 连乘 → ∞（梯度爆炸）

    6.2 解决方案汇总¶

    问题 解决方案
    梯度消失 ReLU/LeakyReLU(替代Sigmoid); BatchNorm/LayerNorm; 残差连接(ResNet); LSTM/GRU(替代SRN); Kaiming/Xavier初始化
    梯度爆炸 梯度裁剪(clip_grad_norm); 权重正则化(L2/Weight Decay); 降低学习率; BatchNorm

    6.3 梯度裁剪代码¶

    # PyTorch
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=5.0)

# PaddlePaddle
paddle.nn.ClipGradByNorm(clip_norm=5.0)

    ◆ 自测题 ◆¶

    Q1. 如何判断训练中发生了梯度爆炸？

    Q2. 如何判断训练中发生了梯度消失？

    Q3. 为什么 ResNet 的残差连接能缓解梯度消失？

    Q4. 梯度裁剪的 max_norm=5.0 意味着什么？

    答案

    A1. Loss 突然飙升或变成 NaN；梯度范数（gradient norm）异常大（如 >1000）；模型参数突然变成非常大的值。

    A2. Loss 下降极慢或停滞；靠近输入层的权重几乎不更新（梯度值 ≈ 0）；深网络的浅层参数与初始值几乎无变化。

    A3. 残差连接：H(x) = F(x) + x。反向传播时 ∂H/∂x = ∂F/∂x + 1，梯度中"1"的存在保证了即使 ∂F/∂x 很小，梯度也不会消失。从链式求导看，残差连接提供了梯度的"高速公路"。

    A4. 如果所有参数的梯度范数（L2 范数）超过 5.0，则按比例缩放到 5.0。即新的梯度 = 原梯度 × (5.0 / ||grad||)。

    七、初始化方法¶

    方法 适用场景 分布
    零初始化 ❌不应使用 W=0→所有神经元学相同特征
    随机初始化 可用但需小心 N(0, 0.01)
    Xavier (Glorot) Sigmoid/Tanh U(−√(6/(fan_in+fan_out)), √(...))
    Kaiming (He) ReLU N(0, √(2/fan_in)) 
    # PyTorch 中手动初始化
nn.init.xavier_uniform_(self.fc.weight)
nn.init.kaiming_normal_(self.conv.weight, mode='fan_out', nonlinearity='relu')

    ◆ 自测题 ◆¶

    Q1. 为什么不能用全零初始化？

    Q2. Kaiming 初始化的设计原理是什么？为什么配合 ReLU 使用？

    答案

    A1. 全零初始化导致所有神经元在前向传播时产生相同的输出，反向传播时得到相同的梯度，所有神经元学习相同的特征（对称性问题），等价于每层只有一个神经元在学。

    A2. Kaiming初始化将权重的方差设为 2/fan_in。因为 ReLU 将一半的神经元输出置0（方差减半），乘以2补偿这一效应，使得通过ReLU后输出的方差保持稳定，在深层网络中前向和反向传播的数值不收缩也不膨胀。

    八、评估指标¶

    指标 公式 适用
    准确率 正确数/总数 类别均衡的分类
    精确率(Precision) TP/(TP+FP) "预测为真的中有多少是真的"
    召回率(Recall) TP/(TP+FN) "所有真的中被找出多少"
    F1 2·P·R/(P+R) 精确率和召回率的调和平均
    混淆矩阵 — 可视化每类预测情况
    AUC-ROC — 二分类排序质量

    ◆ 自测题 ◆¶

    Q1. 一个模型把100张图中99张预测为猫(实际90张猫)，精确率和召回率各是多少？

    Q2. 什么情况下准确率高但F1很低？

    答案

    A1. 预测猫: 99张，其中真猫90张 → TP=90, FP=9。总真猫: 100张含90猫+10狗 → FN=0(猫全检出)。Precision=90/99≈90.9%, Recall=90/90=100%。

    A2. 类别严重不均衡时。例如1000张图，990张猫，10张狗；模型全部预测为猫，准确率=99%，但狗的F1=0（因为Recall=0）。

    九、Numpy/PyTorch 高频操作¶

    9.1 维度变换¶

    # PyTorch
x = torch.randn(2, 3, 224, 224)   # (B, C, H, W)

x.view(2, -1)               # (2, 3*224*224)  展平但保持batch
x.permute(0, 2, 3, 1)       # (2, 224, 224, 3) 转HWC格式
x.transpose(1, 2)           # (2, 224, 3, 224) 交换C和H
x.unsqueeze(0)              # (1, 2, 3, 224, 224) 增加维度
x.squeeze(0)                # 去掉size=1的维度

    9.2 view vs transpose vs permute¶

    操作 是否改变内存布局 使用场景
    view 不改变（要求连续内存） 改变 shape，如 Flatten
    transpose 改变 stride 交换两个维度
    permute 改变 stride 多个维度重排

    考试必考：view 需要连续内存，transpose 后需加 .contiguous() 才能 view

    9.3 require_grad 和 no_grad¶

    # 训练时
x = torch.randn(10, requires_grad=True)
y = model(x)
loss = criterion(y, target)
loss.backward()  # 自动计算所有requires_grad=True的张量的梯度

# 推理时
with torch.no_grad():      # 不计算梯度，节省显存
    output = model(x)

model.eval()               # 切换评估模式（影响Dropout、BN等）

    ◆ 自测题 ◆¶

    Q1. x.view(2, -1) 中 -1 的含义？

    Q2. 以下代码会报错吗？为什么？ 

    x = torch.randn(2, 3, 4)
y = x.transpose(0, 2)
z = y.view(8, 3)

    Q3. model.eval() 和 torch.no_grad() 的区别？

    答案

    A1. -1 表示自动推断该维度的大小，确保总元素数不变。如 x 有 2×3×224×224=301056 个元素，x.view(2, -1) → (2, 150528)。

    A2. 会报错。transpose 改变了 stride（不连续），直接 .view() 需要连续内存，应先 .contiguous():

    ```python z = y.contiguous().view(8, 3) ```

    A3. model.eval() 改变模型行为（Dropout关闭，BN用全局统计量）。torch.no_grad() 关闭自动求导，节省显存。推理时两者通常都使用。

    ◆ 自测题 ◆¶

    Q1. 你要做一个项目：根据昨天的股票价格预测明天的价格，数据量1000条。选什么模型？为什么？

    Q2. 你要识别CT影像中的肿瘤区域（需要像素级标注），选什么模型？

    Q3. BatchNorm 在 batch_size=2 时效果差，应该换成什么？

    答案

    A1. LSTM 或 GRU。时序数据+小数据集，不适合 Transformer（需要大量数据）。LSTM 门控机制适合捕获时间依赖。也可以用简单的一维 CNN。

    A2. U-Net。Encoder-Decoder 结构 + 跳跃连接，适合医学图像分割。输入为CT图像，输出为每个像素的肿瘤/非肿瘤标签。

    A3. 换成 LayerNorm 或 GroupNorm。BN 的统计量在 batch=2 时极不稳定，训练和推理差异大。LN对batch_size不敏感。
