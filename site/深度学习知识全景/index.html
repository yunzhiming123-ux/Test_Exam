    深度学习知识全景¶

    不按教材讲，按知识发展的内在逻辑讲 每条主线：问题起源 → 早期方案 → 发现新问题 → 新解决方案 → 又引出什么问题…… 结合 PyTorch 代码实战 + 每节随堂练习

    第一章：从线性模型到深度神经网络¶

    1.1 线性回归——一切的起点¶

    问题起源（1950年代）：我们想用数学模型预测一个连续值（比如房价）。最简单的方案是用一条直线拟合数据。

    方案：$\hat{y} = Wx + b$

    代码（PyTorch）： 

    import torch.nn as nn

class LinearRegression(nn.Module):
    def __init__(self, in_features):
        super().__init__()
        self.linear = nn.Linear(in_features, 1)

    def forward(self, x):
        return self.linear(x)

    怎么看代码： - nn.Linear(in, out) = 创建参数矩阵 W(in×out) 和偏置 b(out) - forward() 定义数据流向，PyTorch 自动计算反向传播 - 损失函数用 MSELoss：criterion = nn.MSELoss()

    问题暴露：线性模型只能画直线/平面，无法拟合复杂曲线（如 XOR 问题，Minsky 1969 证明感知机无法解决）。

    1.2 激活函数——打破线性的关键¶

    解决方案：在线性变换后加入非线性激活函数。

    $$z = Wx + b \quad \rightarrow \quad a = f(z)$$

    发展脉络：

    时代 函数 公式 优点 问题
    1958 Step f(z)=1 if z>0 简单 不可导
    1960-80s Sigmoid $\frac{1}{1+e^{-x}}$ (0,1) 可导 梯度消失 $\sigma' \le 0.25$
    1990s Tanh $\frac{e^x-e^{-x}}{e^x+e^{-x}}$ (-1,1) 零均值 两端仍饱和
    2012+ ReLU $\max(0,x)$ [0,∞) 梯度无损、计算快 Dead ReLU
    2015+ LeakyReLU $\max(0.01x,x)$ 解决Dead ReLU 负区间斜率固定
    2018+ GELU $x \cdot \Phi(x)$ 平滑门控 计算复杂
    2020+ SwiGLU $\text{Swish}(xW_1) \otimes (xW_2)$ LLM首选 两组参数

    核心问题链： 1. Step不可导 → 改用Sigmoid → 梯度消失 → 改用Tanh → 仍有饱和 → 改用ReLU → Dead ReLU → 改用LeakyReLU/GELU

    PyTorch 代码： 

    import torch.nn.functional as F

x = torch.randn(10)
F.relu(x)              # 隐藏层首选
F.leaky_relu(x, 0.01)  # 负区间有小梯度
F.gelu(x)              # Transformer 默认
torch.sigmoid(x)       # 二分类输出层
F.softmax(x, dim=0)    # 多分类输出层

    1.3 多层感知机（MLP/DNN）¶

    发展历程： 1. 单层感知机(1958) → 只能线性分类 → Minsky 证明 XOR 不行 2. BP算法+多层(1986) → 理论万能近似 → 但梯度消失训不好 3. 预训练+微调(2006, Hinton) → DBN 逐层预训练 → DNN 复兴 4. ReLU+BN+残差(2012-2015) → 彻底解决训练问题 → 深度学习胜利

    参数量计算（必考）： $$\text{Params} = D_{in} \times D_{out} + D_{out}(\text{bias})$$

    class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)   # 200,960
        self.fc2 = nn.Linear(256, 128)   # 32,896
        self.fc3 = nn.Linear(128, 10)    # 1,290
        # 总计: 235,146

    def forward(self, x):
        x = x.view(x.size(0), -1)  # 展平
        x = F.relu(self.fc1(x))
        x = F.relu(self.fc2(x))
        return self.fc3(x)  # 输出logits，不加激活

    怎么看代码： - view(x.size(0), -1)：batch 维保持，其余展平 - 每层 Linear → ReLU 是标准模式 - 最后一层不加激活 → 输出 logits → 配合 CrossEntropyLoss

    DNN 处理图像的致命问题 → 由此引出 CNN： - 参数量爆炸（224×224×3 展平 = 150,528 维） - 丢失空间结构 - 无平移不变性

    ◆ 随堂练习 ◆¶

    Q1（选择题）. 以下激活函数中，哪个最适合深层网络隐藏层？ A. Sigmoid B. Tanh C. ReLU D. Softmax

    答案 **C**。ReLU 正区间导数=1，梯度无损传播。Sigmoid 导数 ≤0.25 导致梯度消失。Softmax 只用于输出层。

    Q2（计算题）. 一个3层DNN：Linear(784→256)→ReLU→Linear(256→128)→ReLU→Linear(128→10)，含bias，总参数量？

    答案 Layer1: 784×256+256=200,960; Layer2: 256×128+128=32,896; Layer3: 128×10+10=1,290; **总计=235,146**

    Q3（简答题）. 为什么回归任务的输出层不加激活函数？

    答案 回归需要输出任意实数值。加Sigmoid会限制在(0,1)，加ReLU限制在[0,∞)，都无法预测负值或超出范围的值。

    第二章：卷积神经网络（CNN）——解决图像问题¶

    2.1 为什么需要CNN？¶

    问题：DNN处理图像有三大致命问题（见1.3节末尾）。

    解决方案（1998, LeCun → 2012, Krizhevsky）：用局部连接+权重共享替代全连接。

    DNN CNN
    连接方式 每个神经元连所有输入 卷积核只看局部区域
    参数量 150528×hidden = 百万级 3×64×3×3 = 1728（第一层）
    空间结构 展平丢失 保持 H×W
    平移不变性 无 同一卷积核滑动全图

    2.2 卷积运算¶

    本质：K×K 卷积核在图像上滑动，逐元素乘法再求和 → 得到特征图上一个值。

    输出尺寸公式（必考）： $$H_{out} = \left\lfloor \frac{H_{in} - K + 2P}{S} \right\rfloor + 1$$

    参数量公式： $$\text{Params} = C_{in} \times C_{out} \times K^2 + C_{out}(\text{bias})$$

    代码： 

    self.conv = nn.Conv2d(3, 64, kernel_size=3, padding=1, stride=1)
# in_channels=3(RGB), out_channels=64(64种特征检测器)
# kernel_size=3(3×3窗口), padding=1(same padding保持尺寸)
# 参数量: 3×64×9+64 = 1792

    怎么看代码： - nn.Conv2d(in, out, K)：in 个输入通道，out 个卷积核，每个核 K×K - padding=1 配合 K=3 → same padding → 输出尺寸 = 输入尺寸 - stride=2 → 下采样，输出尺寸减半

    卷积核学到什么？ - 浅层：边缘、颜色梯度 - 中层：纹理、形状轮廓 - 深层：物体部件、语义特征

    2.3 池化与下采样¶

    问题：特征图越来越大，计算量爆炸。需要减小空间尺寸。

    方法 操作 参数 特点
    MaxPool2D(2,2) 2×2取最大值 0 保留最显著特征，最常用
    AvgPool2D(2,2) 2×2取平均值 0 保留整体信息
    stride>1卷积 步长2的卷积 有 可学习的下采样
    AdaptiveAvgPool2D(H,W) 自适应 0 接受任意输入
    GlobalAvgPool 整图平均→1值 0 替代Flatten，无参数

    2.4 CNN经典架构演进¶

    LeNet(1998): Conv→Pool→Conv→Pool→FC
    ↓ 问题：太浅，学不到复杂特征
AlexNet(2012): 8层+ReLU+Dropout+GPU
    ↓ 问题：层数不够深
VGG(2014): 19层，全部3×3小核
    ↓ 问题：太深梯度消失，FC参数爆炸
GoogLeNet(2014): Inception多尺度并行
    ↓ 问题：结构复杂不好调
ResNet(2015): 152层+残差连接
    ↓ 突破：跳跃连接解决深度训练

    2.5 残差连接——突破深度限制（核心）¶

    问题：VGG 最深只有19层，因为更深的网络会出现梯度消失和退化问题。

    解决方案（ResNet, 2015）：跳跃连接

    $$H(x) = F(x) + x$$

    为什么有效：反向传播时 $\frac{\partial H}{\partial x} = \frac{\partial F}{\partial x} + \mathbf{1}$，+1 保证梯度至少为1。

    class BasicBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, 3, padding=1)
        self.bn1 = nn.BatchNorm2d(channels)
        self.conv2 = nn.Conv2d(channels, channels, 3, padding=1)
        self.bn2 = nn.BatchNorm2d(channels)

    def forward(self, x):
        identity = x
        out = F.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out += identity  # 关键！F(x) + x
        return F.relu(out)

    Bottleneck 结构（ResNet50+）： 

    1×1 Conv (256→64) → 3×3 Conv (64→64) → 1×1 Conv (64→256)
参数量: 256×64 + 64×64×9 + 64×256 = 69K
vs 直接 3×3: 256×256×9 = 590K (节省 1/8!)

    2.6 迁移学习——站在巨人肩膀上¶

    问题：训练CNN需要大量数据和计算资源。

    方案：在ImageNet上预训练的模型已学到通用视觉特征，可直接复用。

    import torchvision.models as models

model = models.resnet18(pretrained=True)
model.fc = nn.Linear(model.fc.in_features, num_classes)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-4)
    策略 做法 适用
    特征提取器 冻结预训练权重，只训分类头 小数据 (<1000)
    微调 全部参数用小lr训练 中等数据

    ◆ 随堂练习 ◆¶

    Q1（计算题）. 输入32×32×3, Conv2D(3,64,3,padding=1,stride=1) → 输出尺寸？参数量？

    答案 输出: (64, 32, 32)。参数量: 3×64×9+64 = **1792**。

    Q2（选择题）. 为什么常用3×3卷积而很少用7×7？ A. 3×3计算更快 B. 2层3×3感受野=5×5但参数更少且多一层非线性 C. 7×7无法提取特征 D. 3×3的padding更容易

    答案 **B**。3层3×3感受野=7×7，参数 $3 \times 9C^2 = 27C^2$，而1层7×7需要 $49C^2$，参数近2倍；且中间多了2个ReLU增加非线性。

    Q3（简答题）. ResNet为什么能堆叠到152层而VGG只有19层？

    答案 残差连接 $H(x)=F(x)+x$，反向传播时 $\partial H/\partial x = \partial F/\partial x + 1$。+1保证即使深层梯度趋近于0，梯度也至少为1。VGG没有跳跃连接，梯度必须经每层激活函数逐层回传，连乘导致浅层梯度消失。

    Q4（简答题）. Global Average Pooling 相比 Flatten + FC 的优势？

    答案 ①零参数→不会过拟合；②输出维度=通道数（固定），不依赖输入尺寸→可接受任意大小图像。

    第三章：序列模型——RNN/LSTM/GRU¶

    3.1 为什么需要序列模型？¶

    问题：CNN和DNN的输入大小固定，每次处理独立样本。但很多数据是序列：文本、语音、股票、视频。

    → 需要一种能"记住"之前信息的网络 → RNN

    3.2 SimpleRNN¶

    核心公式： $$h_t = \tanh(W x_t + U h_{t-1} + b)$$

    关键特征： - 权重共享：W、U、b 在所有时间步复用 → 无论序列多长参数量不变 - 循环：$h_t$ 同时依赖 $x_t$ 和 $h_{t-1}$ → 信息跨时间传递

    rnn = nn.RNN(input_size=128, hidden_size=256, num_layers=1, batch_first=True)
# 输入: (B, T, 128) → 输出: (B, T, 256)

    问题暴露：梯度消失/爆炸！

    3.3 BPTT与梯度问题¶

    $$\frac{\partial L}{\partial h_1} = \frac{\partial L}{\partial h_T} \cdot \prod_{t=2}^{T} \frac{\partial h_t}{\partial h_{t-1}}$$

    • $|\partial h/\partial h| < 1$ → 连乘 → 0 → 梯度消失
    • $|U| > 1$ → 连乘 → ∞ → 梯度爆炸（loss变NaN）

    3.4 LSTM——用门控解决长距离依赖¶

    核心创新：引入细胞状态 $c_t$，其更新接近线性： $$c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$$

    当遗忘门 $f_t \approx 1$ 时 → 梯度无损沿时间步传播！

    六个公式（必考）： 

    遗忘门:   f_t = σ(W_f·x_t + U_f·h_{t-1} + b_f)   → 决定扔掉哪些旧记忆
输入门:   i_t = σ(W_i·x_t + U_i·h_{t-1} + b_i)   → 决定写入哪些新信息
候选记忆: c̃_t = tanh(W_c·x_t + U_c·h_{t-1} + b_c) → 新信息候选值
细胞更新: c_t = f_t ⊙ c_{t-1} + i_t ⊙ c̃_t          → 记忆更新
输出门:   o_t = σ(W_o·x_t + U_o·h_{t-1} + b_o)   → 决定输出什么
隐状态:   h_t = o_t ⊙ tanh(c_t)                   → 当前输出

    共12个参数矩阵：4×{W, U, b} × {遗忘, 输入, 候选, 输出}

    lstm = nn.LSTM(input_size=128, hidden_size=256, num_layers=2,
               batch_first=True, dropout=0.3)
output, (hn, cn) = lstm(x)
# output: (B, T, 256) 每个时间步输出
# hn, cn: (2, B, 256) 最后状态

    3.5 GRU——LSTM的简化版¶

    LSTM GRU
    门控数 3 2
    细胞状态 有(独立) 无(合并到h)
    参数矩阵 12 9
    何时选 大数据/精细控制 小数据/追求速度

    策略 做法 优点 缺点
    Teacher Forcing 用真实标签 训练稳定 Exposure Bias
    Free Running 用模型预测 和推理一致 误差累积

    ◆ 随堂练习 ◆¶

    Q1（填空题）. LSTM的三个门分别用_激活函数，原因是_。

    答案 **Sigmoid**。输出(0,1)天然代表"门"的开闭程度：0=全关，1=全开。

    Q2（简答题）. 从梯度角度解释为什么LSTM比SRN更能缓解梯度消失。

    答案 LSTM细胞状态更新 $c_t = f_t \odot c_{t-1} + ...$，反向传播时 $\partial c_t/\partial c_{t-1} = f_t$（元素级乘法）。当 $f_t \approx 1$ 时梯度无损跨时间步传播。SRN需要每次乘 $\tanh'$ 和矩阵 $U$（连乘导致消失）。

    Q3（计算题）. 单层单向LSTM, input_size=128, hidden_size=256, 不含bias, 参数量？

    答案 4个门×(W: 128×256 + U: 256×256) = 4×(32768+65536) = 4×98304 = **393,216**

    Q4（选择题）. num_layers=2 和 bidirectional=True 的区别？ A. 没有区别 B. num_layers=2纵向堆叠两层；bidirectional=True正向+反向各一层 C. num_layers增加参数，bidirectional不增加 D. bidirectional只影响输出

    答案 **B**。num_layers=2纵向堆叠，输出维度不变，参数翻倍。bidirectional=True正向+反向各一层，输出维度翻倍。

    第四章：注意力机制与Transformer¶

    4.1 RNN的瓶颈 → Attention的诞生¶

    RNN的两个痛点： 1. 串行计算：第t步必须等第t-1步 → 无法并行 → 训练慢 2. 长距离依赖：即使LSTM，超过100步的关系也很难学好

    解决方案（2017, Vaswani）：Self-Attention 让每个位置直接"看到"所有其他位置。

    4.2 Self-Attention¶

    公式（必考）： $$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

    通俗理解： - Q（Query）："我在找什么信息？" - K（Key）："我身上有什么信息？" - V（Value）："我的实际内容是什么？"

    为什么要除以 $\sqrt{d_k}$？ $QK^T$ 的方差 ≈ $d_k$。除以 $\sqrt{d_k}$ 将方差稳定在1，避免Softmax饱和。

    def self_attention(Q, K, V, mask=None):
    d_k = Q.size(-1)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / (d_k ** 0.5)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))
    attn = F.softmax(scores, dim=-1)
    return torch.matmul(attn, V), attn

    复杂度：$O(L^2 \cdot d)$，长序列很慢。

    4.3 Multi-Head Attention¶

    为什么需要多头？ 单头只能关注一种模式。多头让模型同时关注不同关系。

    $$d_k = d_{model} / h$$

    mha = nn.MultiheadAttention(embed_dim=256, num_heads=8, batch_first=True)
# 每个头 d_k = 256/8 = 32

    4.4 Transformer完整结构¶

    Encoder Block（2个子层）： 

    输入 → [Multi-Head Self-Attn] → Add & LN → [FFN] → Add & LN → 输出

    Decoder Block（3个子层）： 

    输入 → [Masked Self-Attn] → Add & LN
     → [Cross-Attn(Q=Dec, KV=Enc)] → Add & LN
     → [FFN] → Add & LN → 输出

    Pre-LN vs Post-LN：

    Post-LN Pre-LN（现代主流）
    公式 LN(x + Sublayer(x)) x + Sublayer(LN(x))
    梯度 需通过LN 残差直通
    训练 需Warmup 无需Warmup

    位置编码： 

    # 正弦编码（可外推任意长度）
PE(pos, 2i) = sin(pos / 10000^(2i/d))
# 可学习编码（BERT/ViT使用）
self.pos_emb = nn.Embedding(max_len, d_model)
x = x + self.pos_emb(positions)

    4.5 自回归生成与解码策略¶

    自回归：$P(x) = \prod_{t=1}^{T} P(x_t | x_1,...,x_{t-1})$

    Causal Mask：下三角矩阵，防止看到未来。

    策略 特点
    Greedy 每步取最大概率，快速但单调
    Beam Search 保留top-K路径，翻译常用
    Top-k采样 从top-k中随机选，增加多样性
    Top-p(Nucleus) 自适应，效果好
    Temperature τ>1→多样，τ<1→确定

    4.6 三种架构对比¶

    架构 代表 注意力 任务
    Encoder-Only BERT 双向 分类、NER
    Decoder-Only GPT 单向Causal 生成、对话
    Encoder-Decoder T5 双向+Cross 翻译、摘要

    4.7 KV-Cache（推理优化）¶

    自回归生成第t步不需要重算前t-1步的KV，缓存历史KV，只算新token。将 $O(t^2)$ 降为 $O(t)$。

    ◆ 随堂练习 ◆¶

    Q1（选择题）. Attention公式中除以 $\sqrt{d_k}$ 的目的是？ A. 增大注意力分数 B. 防止点积过大导致Softmax梯度消失 C. 减小计算量 D. 增加模型复杂度

    答案 **B**。$QK^T$ 的方差随 $d_k$ 线性增长，除以 $\sqrt{d_k}$ 稳定方差。

    Q2（简答题）. Encoder-Only(BERT) vs Decoder-Only(GPT) 的核心区别？

    答案 BERT: 双向Self-Attn + Masked Language Model → 理解任务。GPT: 单向Causal Self-Attn + Next Token Prediction → 生成任务。

    Q3（简答题）. Transformer为什么用LayerNorm而不是BatchNorm？

    答案 ①序列长度可变，BN在序列维度统计不稳定；②推理时可能单样本，BN需要batch统计量；③LN对batch_size无依赖。

    Q4（简答题）. KV-Cache为什么能加速自回归推理？

    答案 缓存已生成token的Key和Value。第t步只需计算新token的QKV，无需重算历史。将 $O(t^2)$ 降为 $O(t)$。

    第五章：训练的核心问题——梯度与正则化¶

    5.1 梯度消失的完整解决方案链¶

    Step函数不可导 → Sigmoid可导但梯度消失(σ'≤0.25)
                 ↓
            用ReLU替代(正区间导数=1)
                 ↓
            Dead ReLU问题
                 ↓
            LeakyReLU/GELU
                 ↓
            深网络仍有梯度衰减
                 ↓
            BatchNorm/LayerNorm(归一化稳定分布)
                 ↓
            残差连接(ResNet)（+1梯度高速通道）
                 ↓
            更好的初始化(Kaiming/Xavier)

    5.2 梯度爆炸的解决方案¶

    现象：loss 突然飙升/变NaN

    方案 做法
    梯度裁剪（首选） clip_grad_norm_(params, max_norm=5.0)
    降低学习率 减小更新步长
    L2正则 限制权重大小
    BatchNorm 稳定激活值分布

    5.3 正则化技术全家¶

    问题：模型太复杂/数据太少 → 过拟合

    技术 原理 适用
    Dropout 训练时随机丢弃神经元→集成多模型 通用
    L1正则 λΣ|W| → 稀疏权重 特征选择
    L2正则/Weight Decay λΣW² → 限制权重大小 通用
    BatchNorm mini-batch噪声→轻微正则 CNN
    数据增强 随机变换增加多样性 图像
    早停 val loss不降就停 总是用
    Label Smoothing 标签不绝对化 防过度自信

    Dropout详解： 

    self.dropout = nn.Dropout(p=0.5)  # 训练时50%丢弃
# train(): 随机丢弃，输出×1/(1-p)保持期望
# eval(): 关闭，全部参与

    过拟合vs欠拟合判断：

    训练集 验证集 说明
    正常 loss低 loss低，接近 ✅
    过拟合 loss很低 loss高，差距大 记住训练集
    欠拟合 loss高 loss也高 模型太弱

    5.4 BatchNorm vs LayerNorm¶

    BatchNorm LayerNorm
    归一化维度 batch feature
    batch依赖 强（小batch差） 无
    适用模型 CNN Transformer/RNN

    5.5 初始化方法¶

    方法 适用 核心
    全零 ❌ 对称问题（所有神经元一样）
    Xavier Sigmoid/Tanh $U(-\sqrt{6/(in+out)}, \sqrt{...})$
    Kaiming ReLU $N(0, \sqrt{2/fan_in})$

    ◆ 随堂练习 ◆¶

    Q1（简答题）. 列出解决梯度消失的5种方法。

    答案 ReLU替代Sigmoid; BatchNorm/LayerNorm; 残差连接; LSTM/GRU替代SRN; Kaiming/Xavier初始化。

    Q2（判断题）. Dropout在训练和测试时行为相同。

    答案 **错**。训练时随机丢弃+缩放，测试时全部参与不缩放。

    Q3（场景题）. train_loss=0.05, val_loss=0.8，这是什么情况？怎么解决？

    答案 严重过拟合。解决：加Dropout/L2正则、数据增强、减小模型、早停、获取更多数据。

    第六章：优化器与学习率调度¶

    6.1 优化器发展脉络¶

    SGD(1950s): θ -= η·∇L
    ↓ 问题：收敛慢，震荡
SGD+Momentum(1964): 加入惯性
    ↓ 问题：学习率需手动调
AdaGrad(2011): 自适应学习率
    ↓ 问题：lr越来越小
RMSProp(2012): 指数移动平均
    ↓
Adam(2014): 动量+自适应 → 默认首选
    ↓ 问题：L2正则效果不好
AdamW(2017): 解耦Weight Decay → Transformer标准

    6.2 各优化器特点¶

    优化器 公式（简化） 适用
    SGD θ -= η·g 追求最佳泛化
    SGD+Momentum v=βv+g; θ-=η·v CNN训练
    Adam m=β₁m+(1-β₁)g; v=β₂v+(1-β₂)g²; θ-=η·m/(√v+ε) 通用首选
    AdamW Adam+解耦WD Transformer

    6.3 学习率调度¶

    # StepLR: 每30epoch衰减×0.1
scheduler = StepLR(optimizer, step_size=30, gamma=0.1)

# Cosine: 平滑衰减（最常用）
scheduler = CosineAnnealingLR(optimizer, T_max=epochs)

# Warmup + Cosine: Transformer标准
# 前N轮lr从小到大，之后Cosine衰减

# ReduceLROnPlateau: 指标不降则减lr
scheduler = ReduceLROnPlateau(optimizer, patience=10)

    6.4 标准训练循环¶

    for epoch in range(num_epochs):
    model.train()
    for x, y in train_loader:
        x, y = x.to(device), y.to(device)
        optimizer.zero_grad()     # ① 清空梯度
        output = model(x)         # ② 前向传播
        loss = criterion(output, y) # ③ 计算损失
        loss.backward()           # ④ 反向传播
        optimizer.step()          # ⑤ 更新参数
        scheduler.step()          # ⑥ 更新学习率

    model.eval()                  # 评估模式
    with torch.no_grad():         # 不计算梯度
        for x, y in val_loader:
            output = model(x)

    ◆ 随堂练习 ◆¶

    Q1（判断题）. Adam 比 SGD 收敛快，但精心调参后 SGD 泛化可能更好。

    答案 **正确**。Adam自适应学习率收敛快，但SGD在精细调参后通常能找到更平坦的最优解。

    Q2（简答题）. Transformer 训练为什么需要 Warmup？

    答案 训练初期参数随机，梯度大且不稳定。直接用大lr会导致模型跑偏。Warmup让lr从小逐步增大，先"找到方向"再加速收敛。

    第七章：损失函数选择¶

    7.1 损失函数速查¶

    损失 适用 搭配
    MSE 回归 无激活(Linear)
    MAE 回归（抗异常值） 无激活
    BCEWithLogitsLoss 二分类 Sigmoid(内部)
    CrossEntropyLoss 多分类 LogSoftmax(内部)
    NLLLoss 多分类 需先LogSoftmax

    7.2 CrossEntropyLoss详解（最常考）¶

    # nn.CrossEntropyLoss() = LogSoftmax + NLLLoss

# ✅ 正确：直接传logits
loss = F.cross_entropy(logits, labels)

# ❌ 错误：双重Softmax
output = F.softmax(model(x), dim=1)  # 不要！
loss = F.cross_entropy(output, y)     # 内部又做了！

    症状：loss卡在 $-\log(1/C)$ 不下降（10分类≈2.3026）

    ◆ 随堂练习 ◆¶

    Q1（选择题）. 训练时loss卡在2.3026（10分类），最可能的原因？ A. 学习率太小 B. 双重Softmax C. 数据有问题 D. 模型太简单

    答案 **B**。模型输出做了Softmax又传入CrossEntropyLoss → 双重Softmax → 概率趋于均匀 → loss≈-log(1/10)=2.3026。

    第八章：对比学习与知识蒸馏¶

    8.1 对比学习：无标签也能学表征¶

    核心思想：拉近"相似的"（正样本对），推远"不相似的"（负样本）。

    SimCLR框架： 1. 同一张图做两种增强 → 正样本对 2. 同batch其他图 → 负样本 3. 用InfoNCE损失训练

    $$\mathcal{L} = -\log\frac{\exp(\text{sim}(z_i, z_j)/\tau)}{\sum_k \exp(\text{sim}(z_i, z_k)/\tau)}$$

    关键：训练完后扔掉投影头，只用Encoder做下游任务。

    8.2 知识蒸馏：大教小¶

    核心：大模型（教师）的软标签比one-hot包含更多信息（类间关系）。

    $$\mathcal{L} = \alpha \cdot L_{CE}(student, y) + (1-\alpha) \cdot L_{KL}(student^\tau, teacher^\tau)$$

    ◆ 随堂练习 ◆¶

    Q1（简答题）. 对比学习和知识蒸馏的核心区别？

    答案 对比学习：无标签/自监督，学习"同类相聚、异类分开"的通用表征。知识蒸馏：有教师指导，学生模仿教师的输出分布，目标是压缩模型。

    Q2（简答题）. SimCLR为什么需要大batch_size？

    答案 InfoNCE的负样本来自同batch中其他样本。batch越大→负样本越多→对比信号越强→学习效果越好。原论文用batch=4096。

    Q1（代码分析题）. 分析以下代码： 

    self.conv = nn.Conv2d(3, 64, 3, padding=1)
self.bn = nn.BatchNorm2d(64)
self.pool = nn.MaxPool2d(2, 2)
self.fc = nn.Linear(64 * 16 * 16, 10)
    (1) 这是什么模型？ (2) 输入图像尺寸是多少？ (3) BN应放在什么位置？

    答案 (1) CNN图像分类。(2) 从 `64*16*16` 推断：32×32输入→pool后16×16→64通道。(3) Conv→BN→ReLU→Pool。

    Q2（代码分析题）. 以下代码有什么问题？ 

    x = F.sigmoid(self.fc1(x))

x = self.fc3(x)

    答案 两个问题：①隐藏层用Sigmoid导致梯度消失，应改ReLU。②输出做了Softmax再传入CrossEntropyLoss导致双重Softmax，去掉 `F.softmax`。

    附录：公式速查卡¶

    公式 用途
    $H_{out} = \lfloor(H_{in}-K+2P)/S\rfloor+1$ CNN输出尺寸
    $\text{Params}{Conv} = C{in} \cdot C_{out} \cdot K^2 + C_{out}$ CNN参数量
    $\text{Params}{Linear} = D{in} \cdot D_{out} + D_{out}$ FC参数量
    $\text{Attention}(Q,K,V) = \text{softmax}(QK^T/\sqrt{d_k})V$ 注意力
    $f_t=\sigma(W_fx_t+U_fh_{t-1}+b_f)$ LSTM遗忘门
    $c_t=f_t\odot c_{t-1}+i_t\odot\tilde{c}_t$ LSTM细胞更新
    $h_t = \tanh(Wx_t + Uh_{t-1} + b)$ RNN隐状态
    $\text{CrossEntropy} = -\sum y_c\log(\hat{y}_c)$ 多分类损失
    $\theta_{t+1} = \theta_t - \eta\nabla_\theta L$ 梯度下降
    $\hat{x} = (x-\mu)/\sqrt{\sigma^2+\epsilon}$ BN/LN
    $\sigma'(x) = \sigma(x)(1-\sigma(x)) \leq 0.25$ Sigmoid导数
    $H(x) = F(x) + x$ 残差连接
